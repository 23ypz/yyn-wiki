# ICPC

ICPC，全称 International Collegiate Programming Contest，中文通常称为“国际大学生程序设计竞赛”。它是面向大学生的国际性算法竞赛体系，核心考察算法设计、数据结构、数学推理、代码实现、调试能力和团队协作。

## 竞赛定位

ICPC 更像是大学生算法竞赛的“主线赛事”。对多数高校程序设计队来说，ICPC 区域赛是长期训练的主要目标之一。它的特点是：题目难度跨度大、比赛时间长、队伍协作重要、排名竞争强。

ICPC 不适合只靠临时突击。想在区域赛中取得较好成绩，一般需要长期训练：包括基础算法、数据结构、图论、动态规划、数学、字符串、计算几何、搜索、构造、博弈等内容。

## 参赛对象

ICPC 面向高校学生。一般由同一学校的学生组队参赛，并由教练负责报名和管理。不同赛区、不同年份对参赛资格、学籍年限、参赛次数等会有具体限制，报名时要看当年的区域赛规则。

## 赛制

典型 ICPC 赛制如下：

- 三人一队。
- 通常共用一台电脑。
- 比赛时长通常为 5 小时。
- 题目数量一般较多，区域赛常见为 10 到 13 题左右，具体以当场为准。
- 程序提交后由评测系统自动判断正误。
- 排名通常优先比较 AC 题数，再比较罚时。

罚时的基本理解是：


author submit accepted time penalty

如果一个题最终通过，那么这个题的罚时通常由通过时间和错误提交次数共同决定。直观上，越早通过越好，错误提交越少越好。

!!! note "常见排名规则"
    排名通常先看通过题数；通过题数相同时，再看总罚时。没有通过的题目通常不计罚时，但错误提交会影响该题最终通过后的罚时。具体规则以当场比赛说明为准。

## 获奖率和晋级

ICPC 的获奖和晋级机制和具体赛区强相关，不能简单用一个固定百分比概括。一般来说，校内选拔、网络赛、区域赛、区域决赛、洲际赛、世界总决赛之间有层级关系。

如果以“奖项含金量”粗略理解：

- 区域赛金牌：通常代表非常强的区域竞争力。
- 区域赛银牌：需要较系统的算法训练和稳定发挥。
- 区域赛铜牌：对多数学校来说也已经需要较扎实的训练基础。
- 世界总决赛资格：通常是极高水平目标，竞争非常激烈。

因为每个区域站参赛队伍数量、学校强度、奖牌数量和晋级名额不同，实际获奖率应查看对应区域赛的当年规则与榜单。

## 难度评价

ICPC 难度可以分成三个层面：

### 入门参赛难度

只要学校允许报名并完成校内选拔，入门参赛并不一定很难。很多学校会组织校赛或训练赛，让低年级同学先体验赛制。

### 稳定过题难度

想在正式区域赛中稳定 AC 多题，需要掌握常见算法模板，并具备较强的读题、建模和调试能力。区域赛通常会有签到题、中档题和难题，队伍需要合理分工。

### 获奖难度

获奖尤其是银牌、金牌难度较高。它不仅要求个人能力，还要求队伍配合：谁读题、谁写代码、谁查错、谁负责数学和构造，都需要在训练中形成默契。

## 常见题型

ICPC 题目不会只考模板，而是更重视综合建模。常见方向包括：

- 图论：最短路、生成树、连通性、网络流、匹配。
- 动态规划：线性 DP、区间 DP、树形 DP、状压 DP、概率 DP。
- 数据结构：线段树、树状数组、并查集、堆、平衡结构。
- 字符串：KMP、Z 函数、哈希、Trie、后缀结构。
- 数学：组合计数、数论、概率期望、线性代数思想。
- 搜索与剪枝：DFS、BFS、双向搜索、状态压缩。
- 构造与思维题：找规律、反例、贪心证明。

## 备赛建议

1. 先把语言和基础模板练熟，尤其是 C++，因为很多学校主力队以 C++ 为主。
2. 每周固定参加团队训练，而不是只单人刷题。
3. 赛后复盘比做题数量更重要，要把没过的题补完。
4. 建立队伍分工：有人擅长图论，有人擅长 DP，有人擅长数学和构造。
5. 训练时模拟正式比赛环境，包括一台电脑、打印代码、罚时压力和榜单策略。

## 适合谁参加

ICPC 适合愿意长期投入算法训练的大学生。如果你只是想快速拿一个入门级竞赛经历，ICPC 可能投入较大；如果你希望系统提升算法能力，ICPC 是非常值得长期准备的目标。
