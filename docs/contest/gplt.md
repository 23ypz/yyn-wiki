# 团体程序设计天梯赛

团体程序设计天梯赛通常称为“天梯赛”，全称为“中国高校计算机大赛——团体程序设计天梯赛”。它与 ICPC/CCPC 最大的不同是：天梯赛更强调学校整体队伍的集体得分，而不是三人小队在一台电脑前协作。

## 竞赛定位

天梯赛非常适合高校组织大规模程序设计训练。它既能让基础较弱的同学承担低难度题得分，也能让强选手冲击高难度题，是一种“梯队式”竞赛。

如果 ICPC/CCPC 更像精英小队比赛，那么天梯赛更像学校整体程序设计能力展示。

## 参赛对象

天梯赛面向高校学生。通常由学校统一组织队伍报名，参赛队伍按照组别进行比赛。官方规则中会对本科组、专科组等组别进行划分。

## 赛制

天梯赛的特点是：

- 以学校队伍为单位组织。
- 每名参赛选手独立做题。
- 个人得分会汇总成团队成绩。
- 题目按难度分层，常见为 L1、L2、L3 等梯度。
- 低难度题覆盖基础能力，高难度题区分强选手。

这种赛制下，团队策略和 ICPC 不同：

- ICPC 是三个人共用一台电脑，追求队伍 AC 题数和罚时。
- 天梯赛是多人同时独立作答，追求全队总分最大化。

## 获奖率

天梯赛奖项通常包括个人奖和团队奖，具体比例、等级和组别以当年官方规则为准。由于它是团体赛事，学校整体组织能力会明显影响最终成绩。

相比 ICPC/CCPC，天梯赛的获奖逻辑不只是“一个队伍是否强”，还要看全校队伍中不同水平选手的得分结构。

## 难度评价

天梯赛难度跨度很大：

### L1 难度

主要考察基础语法、模拟、字符串处理、简单数学、输入输出细节。适合刚入门的同学拿分。

### L2 难度

开始考察常见算法和数据结构，例如排序、二分、DFS/BFS、并查集、简单 DP、图论基础等。中等水平选手能在这里拉开差距。

### L3 难度

通常更接近进阶算法题，可能涉及复杂 DP、图论、数据结构综合、数学推导等。强选手能通过 L3 题显著提升团队总分。

## 备赛建议

1. 学校可以按水平分层训练：新手保 L1，中级冲 L2，强手冲 L3。
2. 个人训练时不要只做难题，低难度题的速度和准确率也很重要。
3. 熟练掌握 PTA / OJ 的输入输出和提交习惯。
4. 做题时优先保证确定能拿的分，不要过早卡在难题上。
5. 团队层面要培养稳定得分的“腰部选手”，总分往往不只靠最强几个人。

## 和 ICPC/CCPC 的区别

| 项目 | 天梯赛 | ICPC/CCPC |
| --- | --- | --- |
| 组织形式 | 多人队伍总分 | 三人小队排名 |
| 做题方式 | 每人独立提交 | 队伍协作，共用电脑 |
| 难度结构 | 明显分层 | 整场题目综合分布 |
| 目标 | 学校整体实力 | 小队算法能力和协作 |

## 适合谁参加

天梯赛适合几乎所有阶段的大学生：新手可以从 L1 题开始，中级选手可以冲 L2，高水平选手可以挑战 L3。它尤其适合学校程序设计协会或竞赛队组织集体训练。
