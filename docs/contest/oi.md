# OI 信息学奥赛

OI 是 Olympiad in Informatics 的简称，通常指面向中学生的信息学竞赛体系。它和数学、物理等学科竞赛类似，强调用算法和程序解决抽象问题。

## 适合人群

OI 主要适合中学生，尤其适合希望系统学习算法、数据结构和程序设计的同学。即使不以竞赛升学为目标，OI 的训练方式也很适合打牢算法基础。

## 常见赛事层级

**CSP-J/S**

CSP-J/S 通常被视为信息学竞赛入门和提高阶段的重要训练与认证。J 表示 Junior，偏入门；S 表示 Senior，难度更高。它适合检查基础语法、算法入门、数据结构和综合实现能力。

**NOIP / 省选**

NOIP 和省选通常更接近正式 OI 选拔。题目会更强调综合算法设计，常见内容包括动态规划、图论、数据结构、数论、字符串、搜索和贪心等。

**NOI**

NOI 是国内较高水平的信息学竞赛。题目通常更综合，需要对算法思想、复杂度、数据范围和实现细节都有较强把握。

**IOI**

IOI 是国际信息学奥林匹克竞赛。它面向各国家和地区选拔出的中学生选手，题目通常强调严谨建模和高质量算法设计。

## OI 题目的特点

OI 题目通常不是只考某一个模板，而是把多个知识点组合在一起。一个题目可能表面上是图论，实际还需要动态规划；也可能看起来是数据结构，核心却是数学性质或贪心证明。

很多 OI 题还有部分分设计。也就是说，如果完整做法很难，可以先写暴力、特殊性质或较低复杂度做法，拿到一部分分数。这种模式要求选手不仅会写最优解，也要会根据数据范围设计分层策略。

## 常见知识模块

OI 常见知识模块包括：

- 基础算法：二分、排序、贪心、前缀和、差分。
- 搜索：DFS、BFS、回溯、剪枝、记忆化搜索。
- 动态规划：线性 DP、背包 DP、区间 DP、树形 DP、状压 DP、数位 DP。
- 图论：最短路、最小生成树、拓扑排序、连通性、二分图、网络流。
- 数据结构：并查集、堆、单调栈、树状数组、线段树、ST 表、平衡树思想。
- 数学：快速幂、组合数、逆元、筛法、质因数分解、同余和容斥。
- 字符串：KMP、Z 函数、哈希、Trie、AC 自动机等。

## 训练方式

OI 训练建议遵循“专题学习 → 模板掌握 → 例题理解 → 套题训练 → 赛后复盘”的顺序。

刚开始不要急着追求难题。更重要的是把基础模板写熟，并且理解模板为什么正确。例如学二分时要理解单调性和边界不变量；学线段树时要理解节点维护的区间信息；学 DP 时要能解释状态定义和转移来源。

## 和本知识库的关系

本知识库的大部分算法页面都可以直接服务于 OI 学习。建议先从基础算法、前缀和与差分、简单数据结构和简单 DP 开始，再逐步进入图论、数学、字符串和高级数据结构。
