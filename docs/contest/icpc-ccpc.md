# ICPC 与 CCPC

ICPC 和 CCPC 都是大学阶段非常重要的程序设计竞赛。它们的核心特点是团队协作、限时解题和现场排名。

## ICPC 是什么

ICPC 通常指 International Collegiate Programming Contest，是面向大学生的国际程序设计竞赛。典型形式是三名队员组成一支队伍，在限定时间内合作解决多道算法题。

## CCPC 是什么

CCPC 通常指中国大学生程序设计竞赛。它在训练方式、题目风格和比赛组织上与 ICPC 有很多相似之处，也常被国内高校用作算法竞赛训练和选拔的重要赛事。

## 比赛形式

ICPC/CCPC 常见形式是：

- 三人一队。
- 多道题目，通常需要在数小时内完成。
- 每道题提交程序，由评测系统判断是否通过。
- 排名通常先看通过题数，再看罚时。

这种形式和 OI 的个人赛很不一样。团队不只是三个人分别写题，而是要决定谁读题、谁写代码、谁造样例、谁负责调试、什么时候换题、什么时候放弃某个方向。

## 题目特点

ICPC/CCPC 题目通常覆盖面很广，既有签到题，也有数学、图论、动态规划、数据结构、构造、计算几何、字符串和复杂模拟。题目常常更强调“在现场快速发现可做方向”，而不是只考某个固定模板。

很多题的难点不一定是算法很高级，而是读题复杂、边界细节多、实现容易出错。现场赛中，稳定实现和快速调试非常重要。

## 团队协作

一支成熟队伍通常会有分工，但分工不是固定不变的。常见模式包括：

- 一人优先读全场题，判断难度和可做顺序。
- 一人快速写签到和基础题，保证开局通过题数。
- 一人负责推较难题的思路或数学性质。
- 全队共同检查复杂代码，避免低级错误。

团队赛中最重要的是沟通成本。一个思路如果不能清楚讲给队友，往往很难快速落地成稳定代码。

## 训练建议

ICPC/CCPC 训练不能只刷单题，还需要经常打整场模拟赛。单题训练适合补算法短板，整场训练适合练习策略和协作。

建议训练流程：

1. 平时按专题补算法基础。
2. 每周进行一次整场 VP 或组队训练。
3. 赛后做完整复盘：哪些题读慢了，哪些题想偏了，哪些代码可以模板化。
4. 建立队伍模板库，统一输入输出、图论、数据结构和数学模板。

## 和 OI 的区别

OI 更强调个人完整解决复杂问题的能力；ICPC/CCPC 更强调团队在有限时间内尽可能多地解决问题。OI 中部分分很重要，而 ICPC/CCPC 通常只有通过和未通过两种结果。因此，ICPC/CCPC 中“先拿稳能过的题”非常关键。
