# 刷题面试与数据竞赛

除了传统算法竞赛，还有一些偏面试、工程或数据科学方向的竞赛平台。它们和 OI、ICPC 的训练目标不同，但也很流行。

## LeetCode 周赛和双周赛

LeetCode 的比赛更接近面试题风格。题目常见内容包括数组、字符串、哈希表、栈、队列、二叉树、图、动态规划和贪心。

它的优点是题目长度适中，适合训练快速实现和边界处理。对于准备实习、校招或技术面试的人来说，LeetCode 比传统 OI/ICPC 更贴近面试场景。

训练建议：

- 先按题型刷基础题，熟悉常用数据结构。
- 再参加周赛，训练限时思考和提交速度。
- 赛后重点补第三题和第四题，因为它们通常更能体现算法能力。
- 对常见套路做总结，例如双指针、滑动窗口、单调栈、树形递归和状态压缩。

## Kaggle 数据科学竞赛

Kaggle 更偏数据科学、机器学习和 AI 实验。参赛者通常需要处理数据、构造特征、训练模型、调参、验证方案并提交预测结果。

它和算法竞赛的主要区别是：传统算法竞赛追求确定性程序和复杂度控制；Kaggle 更重视数据理解、实验设计、模型泛化和评估指标。

Kaggle 适合想学习机器学习、数据分析和实际建模流程的人。它不要求你像 OI/ICPC 那样熟悉所有算法模板，但需要较强的 Python 数据处理、统计建模和实验复盘能力。

## 黑客松与工程竞赛

一些高校、企业和社区会举办 Hackathon 或工程类竞赛。这类比赛通常不只看算法，还会看产品设计、工程实现、展示效果、团队协作和创新性。

如果你的目标是项目实践、工程能力或简历展示，这类比赛很有价值。但如果你的目标是提升纯算法能力，仍然应该以 OI、ICPC、Codeforces、AtCoder 等训练为主。

## 如何区分训练目标

如果目标是算法竞赛成绩，优先 OI / ICPC / Codeforces / AtCoder；如果目标是技术面试，优先 LeetCode；如果目标是数据科学和机器学习，优先 Kaggle；如果目标是工程实践，优先 Hackathon。

不同平台之间可以互相补充，但不要混淆训练目标。比如 LeetCode 能提升面试题熟练度，但不一定覆盖 OI 中复杂的数据结构和数学；Kaggle 能提升建模能力，但不等同于算法竞赛训练。
