# 在线算法竞赛

在线算法竞赛平台适合长期训练。它们的优点是比赛频率高、题库丰富、可以虚拟参赛和赛后补题。

## Codeforces

Codeforces 是国际上非常活跃的算法竞赛平台之一。它的比赛通常分为不同难度段，常见形式包括 Div.1、Div.2、Div.3、Div.4、Educational Round 等。

Codeforces 的特点是题目思维性强，很多题并不需要很重的模板，但需要快速观察性质、构造反例和证明贪心或转移。它非常适合训练临场思考能力。

训练建议：

- 新手可以从 Div.4、Div.3 开始。
- 有基础后练 Div.2 A/B/C，逐步提高到 D/E。
- 每场比赛后一定要补题，尤其是赛中差一点做出来的题。
- 复盘时记录“卡住原因”：读题、性质、算法、实现还是边界。

## AtCoder

AtCoder 是日本的在线算法竞赛平台。常见比赛包括 ABC、ARC、AGC 和 AHC。

ABC 适合从入门到中级的训练，题面通常简洁，质量稳定；ARC 和 AGC 更偏思维和较高难度；AHC 是启发式竞赛，更侧重搜索、局部优化、模拟退火、调参和工程实验。

AtCoder 的特点是题目表述清晰，适合系统训练算法基础。对于想稳步提升的人，可以长期按 ABC 的题目顺序训练。

## 洛谷与牛客

洛谷和牛客在中文环境中很常用。洛谷适合查找中文题解、训练 OI 风格题目和整理题单；牛客经常举办高校训练赛、寒暑假集训赛和企业相关比赛，适合国内学生做组队训练和阶段测试。

## Topcoder

Topcoder 是较早的在线算法竞赛平台，历史上对竞技编程发展影响很大。它的 SRM 模式、rating 系统和挑战机制曾经非常有代表性。虽然现在很多训练者更常使用 Codeforces 和 AtCoder，但 Topcoder 的题目和比赛体系仍然有参考价值。

## 在线比赛怎么训练

在线比赛最重要的不是“打了多少场”，而是“每场之后有没有补全”。建议每次比赛后完成下面几件事：

1. 把赛中通过的题复查一遍，看有没有更简洁写法。
2. 把赛中没过但接近的题补出来。
3. 对完全不会的题先看官方题解，再独立重写代码。
4. 把新的算法或技巧整理到知识库对应页面。

## 平台选择建议

如果你想提高综合竞技能力，可以优先 Codeforces；如果你想系统练基础和稳定实现，可以优先 AtCoder；如果你更习惯中文题面，可以用洛谷和牛客辅助训练。最好的方式不是只用一个平台，而是根据阶段选择合适题源。
