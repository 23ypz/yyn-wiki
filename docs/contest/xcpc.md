# XCPC

XCPC 不是某一个单独比赛的官方名称，而是国内算法竞赛圈常用的泛称。它通常指 ICPC、CCPC 以及类似赛制的大学生程序设计竞赛，包括校赛、邀请赛、省赛、区域赛、网络赛、多校训练等。

## 为什么叫 XCPC

因为 ICPC、CCPC、各类 Collegiate Programming Contest 的名字中都有 CPC，所以竞赛圈常用 XCPC 来概括这一类比赛。这里的 `X` 可以理解为不同组织、不同地区、不同层级的程序设计竞赛。

因此，XCPC 更像是一种训练体系，而不是一个具体报名入口。

## 典型赛制

XCPC 风格比赛通常具有这些特点：

- 多题限时。
- 使用在线评测系统。
- 按 AC 题数和罚时排名。
- 题目覆盖算法与数据结构。
- 可能是个人赛，也可能是三人组队赛。
- 高水平正式赛通常更接近 ICPC / CCPC 三人队伍赛制。

## 和 ICPC、CCPC 的关系

可以这样理解：

```text
XCPC 训练体系
├─ ICPC
├─ CCPC
├─ 校赛 / 新生赛 / 选拔赛
├─ 省赛 / 邀请赛
├─ 网络赛 / 多校训练
└─ 各平台 ACM 赛制练习赛
```

ICPC 和 CCPC 是其中最重要、最正式的赛事；而校赛、邀请赛、多校训练则是备赛过程中的重要练兵场。

## 获奖和难度

由于 XCPC 是泛称，所以没有统一获奖率。不同层级差异很大：

- 校内新生赛：更偏入门，题目通常从语法、模拟、简单算法开始。
- 校赛选拔赛：难度中等，可能用于选拔校队成员。
- 省赛/邀请赛：难度明显提高，常见中档算法题。
- ICPC / CCPC 区域赛：高水平，竞争强。
- 多校训练：题目可能接近甚至高于区域赛难度，适合强队训练。

## XCPC 训练重点

XCPC 训练不能只背模板。你需要逐渐建立几个能力：

### 读题建模能力

看到题目后，要判断它可能属于图论、DP、数学、数据结构、字符串还是构造。很多题不会直接告诉你算法，需要自己抽象。

### 复杂度判断能力

数据范围决定算法复杂度。比如：

- $n \le 10^3$ 可能允许 $O(n^2)$。
- $n \le 10^5$ 通常需要 $O(n\log n)$ 或 $O(n)$。
- 状态数较小时可能考虑状压 DP。

### 团队协作能力

三人队伍赛中，队友之间需要分工：

- 谁先扫题。
- 谁写签到题。
- 谁负责复杂题的推导。
- 谁负责查错和构造测试。
- 哪些题该暂时放弃。

### 赛后补题能力

XCPC 训练真正提升来自补题。赛中不会的题，如果赛后也不补，就很难形成知识闭环。

## 适合谁参加

如果你想参加 ICPC 或 CCPC，平时就应该参加大量 XCPC 风格训练赛。它是从“会写算法模板”过渡到“能在比赛中解决问题”的关键训练方式。

## 推荐训练节奏

入门阶段：

```text
语法 + 模拟 + 排序 + 二分 + 前缀和 + BFS
```

进阶阶段：

```text
基础图论 + DP + 并查集 + 线段树 + 字符串算法 + 数论
```

团队阶段：

```text
每周组队赛 + 赛后补题 + 专题训练 + 代码模板整理
```
