# CCPC

CCPC，全称 China Collegiate Programming Contest，中文为“中国大学生程序设计竞赛”。它是国内面向大学生的重要程序设计竞赛，赛制和训练方式与 ICPC 高度接近，是国内高校程序设计队伍的重要目标赛事之一。

## 竞赛定位

如果说 ICPC 是国际主线，那么 CCPC 可以理解为国内大学生程序设计竞赛体系中的重要主线。很多学校会把 ICPC 和 CCPC 放在一起训练，统称为 XCPC 训练体系。

CCPC 的题目风格通常强调算法、数据结构、数学推理和代码实现能力。比赛既考个人能力，也考团队协作和临场策略。

## 参赛对象

CCPC 面向中国高校大学生，通常由学校组织报名。参赛资格、队伍数量、报名名额、是否需要网络赛或校内选拔，会根据当年组委会通知和学校安排确定。

## 赛制

CCPC 通常采用类 ACM / ICPC 赛制：

- 三人一队。
- 限时比赛，常见为 5 小时。
- 使用在线评测系统提交程序。
- 按通过题数和罚时排名。
- 比赛过程中会有实时榜单，封榜后最终排名在赛后公布。

这类赛制的关键不是“写出一个题”那么简单，而是要在有限时间内做出队伍策略：

- 先快速拿下签到题。
- 中期分配不同队员读不同题。
- 对可能卡住的题及时切换。
- 注意罚时，不要在错误思路上无限提交。

## 获奖率

CCPC 各站奖项数量、奖项比例、晋级方式可能根据当年规则、参赛队伍数和站点安排变化。通常可以关注以下几类结果：

- 区域赛金、银、铜奖。
- 邀请赛奖项。
- 女生赛、总决赛或其他专项赛事成绩。
- 学校内部是否认定为 A 类或重点竞赛。

实际获奖比例不能写死，应以 CCPC 官网、当年比赛规则和学校通知为准。

## 难度评价

CCPC 的难度通常高于普通校赛和多数个人入门赛，和 ICPC 区域赛训练体系接近。粗略可以这样理解：

- 入门难度：需要至少掌握基础算法和数据结构。
- 稳定过题难度：需要较强读题、建模、调试能力。
- 获奖难度：需要长期团队训练和稳定发挥。
- 金银牌难度：通常对队伍综合实力要求很高。

## 题目特点

CCPC 题目常见特点是：

1. **题面建模较重要**：很多题不是直接告诉你算法名称，需要从题意抽象出图、序列、集合、状态等模型。
2. **中档题区分度强**：比赛排名往往由中档题决定，而不是只靠签到题。
3. **细节实现要求高**：边界、数据范围、复杂度、long long、取模、浮点误差都可能成为失分点。
4. **团队策略影响结果**：会做题但分工混乱，也可能导致罚时过高或卡题。

## 备赛建议

CCPC 和 ICPC 可以一起准备：

- 每周至少做一次 3 到 5 小时组队训练。
- 建立个人专题分工，比如图论、DP、数学、字符串。
- 复盘时不只看答案，还要整理“为什么当时没想到”。
- 训练时保留代码模板，但不要依赖模板题；真正比赛更常考变形。
- 多参加校赛、邀请赛、牛客多校等，积累临场经验。

## 适合谁参加

CCPC 适合想在国内程序设计竞赛体系中取得成绩的大学生。对于有 ICPC 目标的同学，CCPC 几乎是同一训练路径上的重要比赛。
