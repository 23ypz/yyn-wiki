# ST 表

ST 表适合静态区间查询，例如区间最大值、最小值、gcd 等。它的特点是预处理较重，但查询可以做到 \(O(1)\)。

## 原理详解

设：

\[
st[i][j]=\text{从 } i \text{ 开始，长度为 } 2^j \text{ 的区间最大值}
\]

长度为 \(2^j\) 的区间可以拆成两个长度为 \(2^{j-1}\) 的区间：

\[
[i,i+2^{j-1}-1]
\]

和：

\[
[i+2^{j-1},i+2^j-1]
\]

所以转移为：

\[
st[i][j]=\max(st[i][j-1], st[i+2^{j-1}][j-1])
\]

查询 \([l,r]\) 时，取一个最大的 \(2^k\le r-l+1\)。两个长度为 \(2^k\) 的区间可以覆盖整个查询区间：

\[
[l,l+2^k-1]
\]

和：

\[
[r-2^k+1,r]
\]

最大值、最小值、gcd 允许重复覆盖，所以可以直接取两段的 max/min/gcd。但区间和不可以这样做，因为重复部分会被算两次。

## 代码模板

```python
n = 5
a = [1, 2, 3, 4, 5]
que = [(1, 1), (1, 2), (1, 3), (3, 4), (2, 5)]  # 查询区间最大值
inf = float('inf')

# 预处理log2
bit = [0] * (n + 1)
for i in range(2, n + 1):
    bit[i] = bit[i >> 1] + 1

# 初始化ST表
st = [[-inf] * (bit[-1] + 1) for i in range(n + 1)]
for i, x in enumerate(a, 1):
    st[i][0] = x

# 预处理ST表
for j in range(1, bit[-1] + 1):
    for i in range(1, (n - (1 << j)) + 2):
        st[i][j] = max(st[i][j - 1], st[i + (1 << (j - 1))][j - 1])

# 查询区间最大值
for l, r in que:
    k = bit[r - l + 1]
    print(max(st[l][k], st[r - (1 << k) + 1][k]))
```

## 易错点

- ST 表适合静态数组，不支持高效修改。
- 只有 max/min/gcd 这类重复覆盖不影响结果的运算，才能用两段覆盖查询。
- 预处理循环边界是 `i + 2^j - 1 <= n`。
