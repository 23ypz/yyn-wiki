# 动态规划总览

动态规划（Dynamic Programming，DP）本质上是在做一件事：**把一个大问题拆成若干个会重复出现的小问题，并把小问题的答案保存下来，避免重复计算**。

很多初学者觉得 DP 难，通常不是因为代码长，而是因为不知道状态该怎么定义。比较稳定的分析顺序是：

1. 先想清楚原问题要求什么。
2. 设计一个能描述“局部问题”的状态。
3. 找到最后一步决策，也就是当前状态可以从哪些更小的状态转移而来。
4. 写出边界条件。
5. 确定计算顺序，保证转移时依赖的状态已经算出来。

## 一个通用分析框架

设状态为 `f[...]`。动态规划通常包含下面几个部分。

**状态定义**：`f[...]` 表示什么。状态定义必须足够精确，否则很难写出正确转移。

**状态转移**：当前状态如何由更小的状态得到。例如：

$$
f[i] = \max(f[i], f[j] + value)
$$

这类式子表示：当前状态可以从某个历史状态接过来。

**边界条件**：最小规模问题的答案。例如空序列、单个元素、容量为 0、长度为 1 的区间等。

**计算顺序**：如果 `f[i]` 依赖 `f[j]`，那么必须先算出 `f[j]`，再算 `f[i]`。

## 本章内容

本章按照常见 DP 类型组织：

- **简单 DP**：LIS、LCS 这类线性序列问题。
- **背包 DP**：0-1 背包、完全背包、多重背包。
- **区间 DP**：石子合并、括号匹配、矩阵链乘法这类区间拆分问题。
- **树形 DP**：状态放在树节点或子树上的问题。
- **数位 DP**：按数字的每一位进行搜索和记忆化统计。

学习 DP 时不要只背代码，更重要的是理解“状态为什么这样定义”“转移为什么不会漏”“循环顺序为什么这样写”。

## 常见状态设计方式

### 处理到第几个位置

例如 LIS 中：

$$
f[i] = 	ext{以 } a_i 	ext{ 结尾的最长上升子序列长度}
$$

这种状态常见于线性序列问题。

### 处理两个序列的前缀

例如 LCS 中：

$$
f[i][j] = 	ext{A 的前 } i 	ext{ 个元素与 B 的前 } j 	ext{ 个元素的 LCS 长度}
$$

这种状态常见于两个字符串或两个数组之间的匹配问题。

### 处理容量或资源

例如背包中：

$$
f[j] = 	ext{容量为 } j 	ext{ 时能获得的最大价值}
$$

这种状态常见于“有限资源下做选择”的问题。

### 处理一个区间

例如区间 DP 中：

$$
f[l][r] = 	ext{处理区间 } [l,r] 	ext{ 的最优值}
$$

这种状态常见于合并、消除、匹配和划分问题。

### 处理一棵子树

例如树形 DP 中：

$$
f[x] = 	ext{以 } x 	ext{ 为根的子树中的答案}
$$

这种状态常见于树上选择、树上路径和子树统计问题。

## DP 的常见难点

### 状态不能太大

状态越多，时间和空间复杂度越高。设计状态时要尽量只保留会影响后续决策的信息。

### 转移不能漏

写转移时要从“最后一步”或“当前决策”入手。比如选不选一个物品、是否匹配两个字符、最后一次合并在哪里发生。

### 循环顺序必须正确

递推写法中，当前状态依赖的状态必须提前算好。比如 0-1 背包一维优化时容量要倒序，完全背包一维优化时容量要正序。

### 边界要单独考虑

很多 DP 错误来自边界，例如空区间、容量为 0、字符串长度为 0、叶子节点等。先写清楚边界，再写一般转移，会更稳。

## 学习建议

学习动态规划时，可以按照下面顺序推进：

1. 先掌握 LIS、LCS 这类线性 DP。
2. 再学习背包 DP，理解“选或不选”和滚动数组。
3. 然后学习区间 DP，重点理解“枚举分割点”和“区间长度顺序”。
4. 接着学习树形 DP，理解后序遍历和子树合并。
5. 最后学习数位 DP，理解搜索状态、上界限制和记忆化。
