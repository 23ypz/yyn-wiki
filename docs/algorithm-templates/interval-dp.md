# 区间 DP

区间 DP 是动态规划中比较难的一类，因为它不是“处理到第几个位置”，而是“处理一个连续区间”。这类问题常见于合并、消除、匹配、划分等场景。

判断一个问题是否适合区间 DP，可以重点看三点：

1. 处理对象是不是连续区间，例如子串、子数组、连续石子堆。
2. 大区间能不能由小区间合并得到。
3. 转移时是否需要枚举一个分割点。

## 状态为什么通常写成 f[l][r]

区间 DP 的状态通常是：

$$
f[l][r] = \text{处理区间 } [l,r] \text{ 的最优值或可行性}
$$

这里的“处理”可以是合并、消除、匹配或统计方案。只要一个大区间的结果依赖若干个更短区间，就可以考虑这种状态。

## 转移为什么要枚举分割点

如果区间 $[l,r]$ 的最后一步是由两个相邻区间合并而来，那么最后一次分割点一定是某个 $k$：

$$
[l,k] \quad \text{和} \quad [k+1,r]
$$

所以常见转移是：

$$
f[l][r] = \min_{l\le k<r}\left(f[l][k] + f[k+1][r] + cost(l,r,k)\right)
$$

如果是求最大值，把 `min` 换成 `max` 即可。

## 经典例题：石子合并

!!! example "例题：石子合并"
    有 $n$ 堆石子排成一行，每次只能合并相邻两堆，合并代价为两堆石子的总数。求把所有石子合并成一堆的最小总代价。

设 $f[l][r]$ 表示把第 $l$ 堆到第 $r$ 堆石子合成一堆的最小代价。

如果最后一次合并的分割点是 $k$，那么最后一次合并前，区间已经变成了两堆：

- 左边一堆来自 $[l,k]$；
- 右边一堆来自 $[k+1,r]$。

最后把两堆合起来，额外代价是整个区间的石子总数：

$$
sum(l,r)=a_l+a_{l+1}+\cdots+a_r
$$

因此：

$$
f[l][r]=\min_{l\le k<r}\bigl(f[l][k]+f[k+1][r]+sum(l,r)\bigr)
$$

因为要频繁查询区间和，所以一般配合前缀和：

$$
sum(l,r)=pre[r]-pre[l-1]
$$

## 循环顺序为什么是先小区间再大区间

`f[l][r]` 依赖 `f[l][k]` 和 `f[k+1][r]`。这两个区间都比 `[l,r]` 短。

所以必须先算长度更短的区间，再算长度更长的区间。常见写法有两种：

```python
for length in range(2, n + 1):
    for l in range(...):
        r = l + length - 1
```

也可以让 `l` 从右往左枚举，`r` 从左往右枚举，从而保证依赖状态已经算好。

## 代码模板

下面给出区间 DP 通用模板。模板中的 `tar`、初始值和边界需要根据具体题目修改。

```python
'''
对于类似合法括号字符串（RBS）的消除问题，通常根据题意，会有如下性质：
可以消除相邻的匹配字符。
相邻匹配字符消除后，原本不相邻的字符会变成相邻，可以继续消除。换句话说，设子串 A=x+B+y，如果 x 和 y 是匹配的（可以消除），且子串 B 可以完全消除，那么子串 A 可以完全消除。
设子串 A=B+C，如果子串 B 和 C 可以完全消除，那么子串 A 可以完全消除。
满足上述性质的题目，可以用区间 DP 解决。

定义 f(i,j) 表示消除 s[i] 到 s[j] 的最优值。

根据性质 2，可以把 f(i,j) 缩小成子问题 f(i+1,j−1)。
根据性质 3，可以枚举子串 B 的右端点，即枚举 k=i+1,i+3,i+5,…,j−2，把 f(i,j) 划分成子问题 f(i,k) 和 f(k+1,j)。注意这里枚举 k 的步长是 2，因为每次消除 2 个字符，被消除的子串长度一定是偶数。
边界：f(i+1,i)，即空串。

答案：f(0,n−1)。
'''

from functools import lru_cache

inf = float('inf')


def func1():
    """记忆化搜索解法"""
    n = 10

    @lru_cache(None)
    def dfs(l, r):
        if l == r:
            return 0  # 根据题意修改
        res = inf  # 根据题意修改
        for k in range(l, r):
            tar = 0  # 根据题意修改
            res = min(res, dfs(l, k) + dfs(k + 1, r) + tar)
        return res

    print(dfs(0, n - 1))


def func2():
    """记忆化搜索转递推"""
    n = 10
    # 初始值根据题意修改
    f = [[inf] * n for i in range(n)]
    for i in range(n):
        f[i][i] = 0

    for l in range(n - 1, -1, -1):
        for r in range(l + 1, n):
            for k in range(l, r):
                tar = 0  # 根据题意修改
                f[l][r] = min(f[l][r], f[l][k] + f[k + 1][r] + tar)

    print(f[0][n - 1])
```

## 如何把模板套到题目里

使用区间 DP 模板时，重点改这几处：

1. `f[l][r]` 的含义：是最小代价、最大价值，还是能否消除。
2. 初始值：求最小值通常用 `inf`，求最大值通常用 `-inf`，判断可行性通常用 `False`。
3. 边界：长度为 1 的区间、空区间、长度为 2 的区间是否有特殊值。
4. `tar`：两个子区间合并成大区间时额外产生的代价或贡献。
5. 枚举分割点 `k` 的范围和步长：有些消除类问题只能按偶数长度拆分。
