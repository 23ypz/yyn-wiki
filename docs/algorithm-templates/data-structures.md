# 数据结构总览

数据结构章节整理算法题中常用的维护结构。它们的共同目标是：把“每次都重新枚举”的暴力过程，转化成对某种信息的动态维护。

## 如何选择数据结构

| 需要维护的信息 | 常用结构 | 典型场景 |
| --- | --- | --- |
| 集合连通性、合并关系 | 并查集 | 连通块、等价关系、最小生成树辅助判断 |
| 动态最值或中位数 | 堆 / 对顶堆 / 懒删除堆 | 数据流中位数、滑动窗口、在线最值 |
| 最近更大、更小、窗口最值 | 单调栈 / 单调队列 | Next Greater Element、滑动窗口最大值 |
| 动态有序序列 | SortedList | 排名、第 k 小、前驱后继、动态二分 |
| 前缀和与单点修改 | 树状数组 | 动态区间和、逆序对、计数前缀 |
| 区间查询与复杂区间维护 | 线段树 | 区间最大值、区间和、区间更新 |
| 静态区间查询 | ST 表 | 静态 RMQ、区间 gcd |

## 学习主线

数据结构题通常不是先背模板，而是先判断要维护什么量。例如“区间和”可以用前缀和，但一旦数组会被修改，普通前缀和每次修改都要影响后面大量位置，这时就需要树状数组或线段树。

如果维护的信息可以由左右子区间合并得到，例如最大值、最小值、和、gcd，就可以考虑线段树或 ST 表；如果只需要前缀信息和单点修改，树状数组会更短、更快；如果需要动态有序性，则考虑 SortedList。
